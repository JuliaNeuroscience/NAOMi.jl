I/O

Simulated movies are written to and read from multi-page TIFF stacks via TiffImages.jl. write_tpm_movie is the recommended entry point for saving a finished movie.

NAOMi.write_tiff — Function
write_tiff(path::AbstractString, data::AbstractArray{<:Real};
           dtype::DataType=Float32)

Write a 2-D image or 3-D movie data to a (multi-page) TIFF at path. Each data[:, :, k] becomes one page. Values are converted to dtype (default Float32, matching upstream's single movies). Ports tifwrite.m / tiff_writer.m.

source
NAOMi.read_tiff — Function
read_tiff(path::AbstractString; frames=nothing) -> Array{Float32,3}

Read a TIFF stack at path into an H × W × T array of Float32. frames, if given as a (first, last) tuple or range, truncates to that inclusive page range. Ports tifread.m / tiff_reader.m.

source
NAOMi.write_tiff_blocks — Function
write_tiff_blocks(path::AbstractString, mov::AbstractArray{<:Real,3};
                  blocksize::Integer=2500, dtype::DataType=Float32)
    -> Vector{String}

Write a 3-D movie mov to a series of TIFF files, each holding at most blocksize frames, named stem_00001.tif, stem_00002.tif, … (the extension of path is reused). Returns the list of files written. Ports tifwriteblock.m.

source
NAOMi.write_tpm_movie — Function
write_tpm_movie(path::AbstractString, mov::AbstractArray{<:Real,3};
                blocksize::Integer=2500, dtype::DataType=Float32)
    -> Vector{String}

Write a simulated two-photon movie to disk, dispatching on the file extension of path. Only .tif / .tiff is supported (via write_tiff_blocks); upstream's .fits and .mat branches are not ported. Ports write_TPM_movie.m.
